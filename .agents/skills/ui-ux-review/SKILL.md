# UI UX Review

Use this skill for reviewing interface clarity, accessibility, workflows, visual regressions, and responsive behavior.

## Workflow
- Inspect the actual page and viewport where the issue appears.
- Prioritize usability blockers, broken affordances, inaccessible controls, overlap, and confusing states.
- Preserve established design tokens, spacing, typography, and layout unless fixing a verified UI bug.
- Validate desktop and mobile when the touched UI is responsive.

## Guardrails
- Do not redesign during a bug fix.
- Do not add explanatory clutter to operational dashboards.
- Do not change copy unless the current copy is the verified problem.
